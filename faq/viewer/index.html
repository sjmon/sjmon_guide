Viewer

아래 FAQ 에 없는 문의사항은 담당자에게 직접 연락 바랍니다.

화면이 정상적으로 보이지 않습니다.

Login 화면의 그림이 보이지 않거나, Inventory 메뉴 조회 시 목록이 제대로 조회되지 않는 등, 화면이 제대로 조회되지 않는 경우 브라우저 캐시 가 남아 제대로 보여지지 않을 수 있습니다. Ctrl + F5 혹은 브라우저 설정의 캐시 삭제를 진행한 후 다시 시도해주세요.

Dashboard 의 도넛 차트가 보이지 않습니다.

설치 당일은 데이터가 충분히 수집되지 않아 빈 차트로 보이는 것이 맞습니다.

Manager 설치 위치의 config.properties 에 설정된 db.sched.dashboard.donut.month에 설정된 주기마다 refresh 되므로, 최초 설치 이후에는 그 시간 이후 확인 가능합니다.

db.sched.dashboard.donut.month = 0 1 0 * * ?    # 기본값: 매일 0시 1분 0초

설정 주기 이후에도 계속해서 도넛 차트가 보이지 않는다면 로그를 확인하여 상세 원인을 파악해야 합니다.

Top SQL 의 Trend 차트가 보이지 않습니다.

조회 기간을 1시간 이내로 설정한 경우 차트가 나오지 않습니다. 1시간을 초과한 기간으로 조회한 경우에도 나오지 않는 경우는 연결된 DB Client 가 있는지 확인하거나 로그를 확인하여 상세 원인을 파악해야 합니다.

Viewer 의 시간이 맞지 않습니다. (시계, 차트의 x축, tooltip 등)

OS, DB, Tomcat 의 timezone 이 일치하지 않을 때 일어나는 문제입니다. Viewer 2.1.2 버전에서 패치 완료되었습니다. OS의 Timezone 이 KST일 경우에만 정상적으로 사용 가능합니다.

다만, Tomcat 과 Manager 가 서로 다른 OS에 설치되어 있는 경우에는 Tomcat 실행 옵션에서 -Duser.timezone 옵션을 이용해 Manager에 설치된 OS 의 Timezone 과 통일해야 정상적인 시간 확인 가능합니다.

CentOS 에서 Viewer 접속 시, 위젯 추가 버튼이 동작하지 않습니다.

CentOS의 firefox 와 chrome 에서는 위젯 라이브러리인 gridster 지원 불가로 인하여 해당 기능이 동작하지 않습니다. Viewer 접속은 Windows OS 환경을 권장합니다.

정상 설치 후에도 Viewer 에 접속할 수 없습니다.

방화벽을 확인해야 합니다. service iptables stop이나 systemctl stop firewalld 명령어로 방화벽을 중지한 뒤 다시 시도해주세요.

IE에서 Viewer 에 접속할 수 없습니다.

Manager 와 Tomcat 설치가 정상적으로 완료된 경우, PC 에 설치된 보안프로그램으로 인해 Viewer 접속 시 아무 화면도 나타나지 않을 수 있습니다. 다른 PC나 Chrome 을 통해서는 잘 접속되는지 확인해야 합니다.

Parameter Change History 의 Change History 탭의 결과와, Time-comparative Analysis 탭의 결과가 다릅니다.

Parameter Change History 탭은 전체 시스템 조회가 가능하기 때문에 Manager 시간인 inputtime 기준으로 조회하고, Time-comparative Analysis 탭은 Client 별로 조회하므로 Client 시간인 gettime 기준으로 조회합니다. 따라서, Manager와 Client의 서버 시간이 다른 경우에는 다른 결과가 나올 수 있습니다.

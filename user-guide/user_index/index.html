User Manual

1. Home

HOME은 로그인 후 처음으로 보여지는 메뉴입니다. 우측 상단에서 다크 테마를 해제할 수 있습니다.

전체 메뉴와 모니터 테마를 한 눈에 확인하여 원하는 메뉴로 이동할 수 있습니다.

★ 로 등록한 자주 사용하는 테마 및 메뉴를 확인할 수 있으며, 양측 버튼을 클릭하여 전체 테마와 전체 메뉴를 확인할 수 있습니다.

  • - 유저가 소유한 테마 중 공유하지 않은 테마.
  • - 유저가 소유한 테마 중 공유 중인 테마.
  • - 즐겨찾기 설정하지 않은 테마 및 메뉴
  • - 즐겨찾기 설정하지 않은 테마 및 메뉴
  • - 테마 및 메뉴 새 창으로 열기



2. DASHBOARD

DASHBOARD는 MANAGER에 연결된 CLIENT들의 상태 및 자원 사용 현황, 사용량이 높은 서버, 사용량 패턴, 이벤트 발생 여부 등을 쉽게 확인할 수 있도록 구성되어 있습니다.

DASHBOARD 의 위젯은 한 row 가 다른 row 와 위치 변경이 가능합니다.

1) Notification & Sidebar

모니터 연결 여부와 Event, Process, Error Log, Param 상태를 확인할 수 있습니다.

메뉴 설명
Up List Manager에 연결되어 데이터를 모니터링 할 수 있는 CLIENT를 보여줍니다.
Down List Manager와의 연결이 끊어진 CLIENT를 보여줍니다.
Event List 주의, 경고 권고치 값을 초과한 CLIENT를 보여줍니다.
Process Event List Process Monitoring 중 오류가 있는 CLIENT 정보를 보여줍니다.
Error Log List 에러 로그가 발생한 CLIENT의 정보를 보여줍니다.
Parameter Change List Parameter가 변경된 CLIENT의 정보를 보여줍니다.


해당 메뉴를 통한 모니터링 중 변경이 있을 경우, 카드에 종 모양이 나타납니다. 각 카드를 클릭하면, 우측 사이드바가 나타나면서 상세 리스트를 확인할 수 있습니다.

사이드바에서 원하는 헤더를 클릭하면, 해당하는 상세내용을 확인할 수 있습니다.

Up, Event, Error Log, Param List의 경우에는 서버 클릭 시 관련 메뉴로 이동합니다.

메인 화면 아무 곳이나 클릭하면 사이드바가 닫힙니다.


2) CPU Ranking for the last 5 minutes

최근 5분 동안 CPU 자원에 평균적으로 부하가 많이 발생한 5개의 그룹 정보를 보여줍니다.

차트의 그룹 데이터를 클릭하면, 해당 그룹 내 서버 대상으로 CPU 부하가 많이 발생한 5개의 서버 정보를 보여줍니다.

  • 서버 데이터를 Click하면 아래 위젯에서 해당 CLIENT에 대한 Trend Chart와 실시간 모니터링이 가능합니다.


3) Trend Chart & Realtime Monitoring of the Top Agent

CPU 자원에 부하가 가장 많이 발생한 서버의 트렌드 차트와 실시간 데이터를 모니터링 할 수 있습니다. 랭킹 차트에서 원하는 서버를 클릭한 경우에는, 클릭한 서버에 대한 정보를 보여줍니다.


4) Clock & PM Calendar

매니저의 서버 시간과 라이센스 정보, 사용 용량을 함께 보여주는 시계 위젯을 좌측에서 확인 가능합니다. 우측에는 PM 수집한 날짜와 서버를 확인할 수 있는 달력 위젯을 확인 가능합니다.


5) Overall Performance View

  1. 모니터링 하고자 하는 CLIENT를 선택할 수 있습니다.

  2. 모니터링 하고자 하는 지표를 선택할 수 있습니다.

  3. 주기적으로 변하는 CLIENT의 performance 정보를 확인할 수 있습니다.

  4. 모니터링을 원하지 않는 CLIENT를 삭제할 수 있습니다.

6) Event Today

OPV에 선택된 서버들을 대상으로, 하루에 발생되는 EVENT들에 대해 확인할 수 있습니다. OPV에 아무것도 선택되지 않은 경우에는 전체 서버 Event를 확인할 수 있습니다.

  • EVENT의 시작 시간, 종료 시간을 확인할 수 있으며 이벤트를 CLICK 하면 상세한 EVENT 내용을 보여주는 화면으로 이동하게 됩니다.


7) Target Process Monitoring

OPV에 선택된 서버들을 대상으로, 설정한 Process 에 대한 정보를 실시간으로 Monitoring 할 수 있습니다. OPV에 아무것도 선택되지 않은 경우에는 전체 서버에 대한 정보를 볼 수 있습니다.

  1. 설정값에 해당하는 Process 가 수집되지 않는 경우 CHKREQ (Check Required) Critical 표시됩니다.
  2. 설정값에 해당하는 Process 의 State 가 정상이 아닌 경우 Warning 표시됩니다.
  3. 설정값에 해당하는 Process 의 개수가 1개보다 많은 경우 Count 값에 Bold & Underline 표시됩니다.



3. MONITOR

모니터는 CLIENT들이 가지고 있는 다양한 성능 정보를 자유롭게 배치하여 모니터링 할 수 있도록 해줍니다.
사용자 별로 원하는 성능 정보들을 배치하여 theme로 저장하여 언제든 불러올 수 있도록 도와줍니다.


1) Menu

Monitor는 위젯 추가, Interval 조정, Auto Refresh, Theme, Fixed Host 기능으로 구성되어 있습니다.


STEP 1. 위젯 추가 : 원하는 위젯을 추가할 수 있습니다.
STEP 2. Auto Refresh : 주기적인 Auto Refresh가 발생하고, Checkbox해제 시 해당 화면은 더 이상 Auto Refresh가 안되는 상태로 멈춰 있게 됩니다.
STEP 3. Fixed a System : 기존에 보고 있던 Monitoring화면의 STATISTICS를 “Fixed a System”에서 선택한 CLIENT로 변경합니다.
STEP 4. History Theme : 저장한 Theme List 를 조회 및 선택하여 기존 Theme를 불러올 수 있습니다.
STEP 5. Theme : Monitor화면에서 자유롭게 배치한 widget들이 상태를 Theme로 저장하여, 나중에 조회가 가능합니다.

  • - 테마를 추가 합니다.
  • - 현재 화면에 있는 위젯을 저장합니다.
  • - 테마명을 수정합니다. (자신이 생성한 경우에만 가능)
  • - 테마를 삭제 합니다. (자신이 생성한 경우에만 가능)


2) Screen widgets

Screen Widget을 통해 원하는 chart형식으로 STATISTICS를 조회할 수 있습니다.


STEP 1 : WIDGET 종류에 따라 구분할 수 있는 TAB을 제공하고, WIDGET종류는 CHART, GRID, GROUP, NETWORK, XLOG가 있습니다.
STEP 2 : DEFAULT로 All을 통해 모든 CLIENT를 선택할 수 있고, OS, DB, Network별 CLIENT 선택도 가능합니다.
STEP 3 : SYSTEM을 선택합니다. Host, DB, Application를 선택할 수 있습니다. 그룹을 선택하면 해당 그룹에 포함된 System만 확인 및 선택할 수 있습니다.


STEP 4 : CLIENT에 부합하는 STATISICS를 선택합니다.

  • - 평균 STATISTICS들을 선택할 수 있습니다.

  • - 개별 STATISTICS들을 선택할 수 있습니다.

  • - Process STATISTICS들을 선택할 수 있습니다.

  • - Application STATISTICS들을 선택할 수 있습니다.

3) Widget Menu

MENU를 통해 최대값, 평균값, 현재값 확인 및 알림설정, Report export 등을 할 수 있습니다.


STEP 1 : 현재값, 평균값, 최대값을 라벨에서 확인할 수 있습니다.

  • - 현재 값과 현재 Agent 시간을 확인할 수 있습니다.

  • - Widget 시작 시간부터 평균값을 확인할 수 있습니다.

  • - 최대값과 Agent 시간을 확인할 수 있습니다.

STEP 2 : 차트 관련 설정을 할 수 있습니다.

  • - 해당 위젯의 EVENT 알림을 on/off 합니다.
  • - 해당 위젯의 Type, STATISTICS, System을 수정합니다.
  • - 해당 위젯을 화면에서 삭제합니다.

STEP 3 : 차트 데이터의 Excel EXPORT, PNG Image Download 및 Print를 할 수 있습니다.

4) XLog Chart

application의 트랜잭션이 하나의 점으로 표시 됩니다.


STEP 1 : 위젯 선택 시 XLOG 탭을 클릭합니다.
STEP 2 : SYSTEM과 STATISTICS을 선택합니다.


STEP 3 : 원하는 구간을 드래그 합니다


STEP 4 : 구간 속 XLOG 상세 리스트가 팝업으로 표시 됩니다. 한 개를 선택하면 상세 PROFILE을 확인할 수 있습니다.



4. Tool

1) Summary

OS와 DB에 대하여, 일정 기간 동안의 CPU 사용량 평균 그래프를 조회하여 비교해 볼 수 있습니다.

조회 후 기본 그래프는 CPU 평균 그래프를 보여줍니다. CPU 최대 사용 값과 CPU 평균 사용률에 대해서 비교가 가능합니다.


  • 상세 사용 방법

상단의 메뉴를 통해 아래와 같이 CLIENT와 조회 기간을 선택할 수 있습니다.


STEP 1 : System 선택(OS, DB, aPPLICATION)
STEP 2 : 편리한 방식의 combo방식 기간 선택(1일, 1주일, 1개월, 3개월, 6개월, 1년)
STEP 3 : 사용자가 직접 기간 선택
STEP 4 : all day 선택시 00시 ~ 23시로 자동 선택
STEP 5 : 3번에 선택된 기간에 대한 시간 선택
STEP 6 : TOP 서버 중 N개의 서버 조회

CPU 최대 값과 함께 CPU평균 및 가중치 평균에 대해서 비교할 수 있습니다.

  • TOP 서버들의 CPU 평균

  • TOP 서버들의 CPU 가중치 평균값 확인

※ DB 의 경우 , DB Time을 비교할 수 있습니다.
※ Application 의 경우 , TPS를 비교할 수 있습니다.

특정 CLIENT의 상세내역을 조회하기 위해서는 해당 서버의 BAR를 클릭하면, 해당 서버의 CPU, Memory, Paging, Disk, Network에 대한 사용률을 확인할 수 있습니다.
TOP-DOWN 방식으로 분석이 가능하고, OS는 Top process를 확인할 수 있고 DB는 TOP SQL을 확인할 수 있습니다.
Application은 TOP Service, TOP SQL, Top Exception을 확인할 수 있습니다.

Top Process, Disk, Network TAB을 통해서 가장 많이 사용한 자원을 확인할 수 있습니다.


2) Analyzer

Analyzer기능을 통해 기간별로 CLIENT의 과거 데이터 분석이 가능합니다. 원하는 CLIENT와 STATISTICS를 선택하여 확인이 가능합니다.

한 그래프에서 마우스를 이동하면, 해당 화면의 모든 차트에서 같은 시간대의 시계열 축이 움직이면서 동 시간대 자원 사용률을 비교할 수 있습니다.

* 상세 사용 방법

시계열, 구간 분석도구 입니다. 아래와 같은 방법으로 데이터를 조회할 수 있습니다.

STEP 1 : System을 선택합니다.
STEP 2 : STATISTICS를 선택합니다.
STEP 3 : 기간을 선택합니다.
STEP 4 : 차트의 FOCUS에 맞는 시간을 보여줍니다.


STEP 5 : 조회된 차트, STATISTICS 및 CLIENT를 변경하여 조회하면 새로 차트가 만들어지며 비교분석을 할 수 있습니다.
STEP 6 : Chart Menu 입니다.

  • - 차트 데이터를 초단위, 분단위, 시간단위, 일단위로 조정할 수 있습니다.

  • - 차트 형태를 라인, 바, 스택으로 바꿀 수 있습니다.

  • - 차트를 기준으로 한 상관계수 Report 를 확인할 수 있습니다.

  • - 조회한 차트를 닫습니다.

  • - 차트 데이터를 엑셀 EXPORT, PNG 파일 다운로드, Print로 출력 할 수 있습니다.

STEP 7 : 화면에 보여지는 모든 차트에서 FOCUS를 같은 시점으로 이동시키면서 비교분석 할 수 있습니다.

※ 해당 그래프에서 원하는 구간을 드래그, CLICK하면 자세한 정보를 확인할 수 있는 POP-UP창이 나타납니다.
※ XLOG 차트에 경우 원하는 구간을 드래그 하면 상세 XLOG 리스트 POP-UP 창이 나타납니다.

  • Performance Report 버튼을 클릭하여, 해당 차트의 데이터를 기준으로 하여 상관도를 분석해 주는 Report 를 확인할 수 있습니다.

  • 해당 차트의 System 과 Statistics 중 기준으로 사용할 데이터를 선택하고, 상관도를 분석하고자 하는 System 을 원하는 만큼 선택하면, Target System 이 가지는 모든 스탯과의 상관도를 분석하여 높은 순으로 10개의 스탯 차트를 확인하고, Report 출력할 수 있습니다.


3) Event

이벤트 조회 및 분석 도구 입니다. 이를 통해 EVENT들의 상태를 한 눈에 확인할 수 있습니다.

* 상세 사용 방법


STEP 1 : EVENT를 조회할 대상 CLIENT를 조회합니다.
STEP 2 : EVENT 값을 조회(선택하지 않을 경우 전체 조회)합니다.
STEP 3 : 기간 선택 후 “INQUIRY” 버튼을 누릅니다.


STEP 4 : TOP EVENT 정보를 확인할 수 있습니다.


STEP 5 : EVENT LIST정보를 확인할 수 있습니다.


4) Quick Parameter Check

전체 시스템 파라미터 권고 적용 여부를 확인하는 도구 입니다. 이를 통해 현재 서버의 파라미터가 권고 값을 따르고 있는지 한눈에 확인이 가능합니다.

* 상세 사용 방법


STEP 1 : AIX에서 제공하는 주요 파라미터의 변경여부를 직관적으로 확인이 가능합니다.

  • BLACK : 사용되지 않는 파라미터
  • RED : 권고치에 맞지 않는 파라미터
  • GREEN : 권고치에 해당하는 파라미터


5) Parameter Health Check

6. NAGIOS

NAGIOS에서 수집한 Host, Service의 정보를 쉽게 확인 할 수 있습니다.
※ 해당 메뉴는 NAGIOS와 연동이 되어야 이용 할 수 있는 메뉴입니다.

1) Host Status

NAGIOS에 등록 된 Host의 상태 및 호스트 별 서비스 상태 현황 및 통계를 확인 할 수 있습니다

SJMON Viewer에 접속 -> Nagios -> Host Status 선택

  • 화면의 데이터가 갱신 된 시간을 표기합니다.

  • Host 서버의 UP, Down 상태를 표기합니다.

  • SJMON CLIENT의 실행여부를 표기합니다.

  • 컬럼 값이 'Y’ 일 경우(=실행 상태) 클릭 시 해당 Host의 모니터 화면으로 이동합니다.

  • 컬럼 값이 ‘N' 일 경우(=중지 또는 미 연결 상태 ) 클릭이 불가능합니다.

  • 해당 Host의 모니터 화면으로 이동이 불가합니다

  • 각 호스트의 서비스 상태 별 합산 정보를 표기합니다.
    (상태에 대한 기준은 NAGIOS Rule Configuration에 의해 산정됩니다.)

  • 합산 정보에 대한 셀을 클릭 시 클릭 한 상태의 상세 정보를 하단에 출력합니다.
    (추가 클릭 시 상세 화면은 닫힙니다.)


2) Service Status

NAGIOS에 등록 된 그룹별 Service의 상태 및 서비스 현황을 확인 할 수 있는 메뉴를 제공합니다.
※ 해당 메뉴는 NAGIOS와 연동이 되어야 이용 할 수 있는 메뉴입니다.

SJMON Viewer에 접속 -> Nagios -> Service Status 선택

  • ALL SERVICE STATUS 는 전체 서비스에 대한 상태 별 합계를 나타냅니다.
    (상태에 대한 기준은 NAGIOS Rule Configuration에 의해 산정됩니다)

  • 각 서비스 그룹 별 서비스상태 (좌측부터 OK, WARNING, CRITICAL) 및 합계를 나타냅니다.

  • 상태 도넛 차트를 클릭 시 해당 그룹, 상태에 대한 전체 Service 내역을 팝업으로 확인 할 수 있습니다.


3) Event Console

NAGIOS에 발생한 최근 3일의 모든 이벤트(=HOST, SERVICE 상태 변동 및 NOTIFICATION 발생내역)를 확인 할 수 있습니다.
※ 해당 메뉴는 NAGIOS와 연동이 되어야 이용 할 수 있는 메뉴입니다.

SJMON Viewer에 접속 -> Nagios -> Event Console 선택

  • - 클릭 시 화면을 갱신합니다.

  • Comment 의 SJMON_THRESHOLD로 표기된 내용은 SJMON THRESHOLD로 인하여 발생한 NOTIFICATION 내용입니다.

그 외 원하는 파일을 업로드하고자 하는 경우, 원하는 위치를 작성하여 업로드하면 해당 위치에 저장됩니다.

  • 현재 AIX 버전에서 현재 설치되어 있는 GPFS버전이 지원이 안되는 경우 Unverified로 표시

* VIOS Version Check

  • 해당 장비에서 현재 설치되어 있는 VIOS버전이 지원이 안되는 경우 Unverified로 표시

  • 현재 장비에서 지원이 안되는 경우 Unverified로 표시

3) System Group

CLIENT들의 GROUP을 관리할 수 있습니다. 다양한 메뉴에서 사용됩니다.

4) Threshold

자원 별 임계치 설정을 통해, 특정 임계치를 초과하는 경우 Dashboard, Monitor, Top Menu화면에 표시합니다. 이를 통해 EVENT를 빠르게 찾아서 조치할 수 있도록 도와줍니다.
Nagios 연동 시 Nagios의 자원 별 임계치를 설정하는 Nagios Host & Service Parameter 그리고 Notification 기능을 관리 사용 가능합니다.

4.1) SJMON THRESHOLD 기능

  • - SJMON System에 대하여 실시간 THRESHOLD를 설정하는 화면입니다.

  • - NAGIOS의 RULE, PARAMETER를 설정하는 화면입니다.

  • - NAGIOS의 NOTIFICATION을 설정하는 화면입니다.

  • - 화면 보여주기 / 숨기기

  • - 등록, 수정, 삭제 버튼

  • - 초기화면으로 이동

NAGIOS THRESHOLD, NOTIFICATION CONFIGRUAION 설정 화면은 NAGIOS 로그인이 안 되었을 경우 화면에 보이지 않습니다.
Add 버튼을 통해 자원 별로 원하는 임계치를 설정할 수 있습니다.

  • Threshold Name : 사용자가 인식할 수 있는 Threshold 이름을 입력합니다.

  • Statistics Name : 원하는 성능 정보를 선택합니다(CPU, Memory etc.).

  • Condition : 원하는 성능 정보의 임계치를 크거나, 같거나 작은 값으로 설정합니다.

  • Threshold Value : 원하는 성능 정보의 임계치를 설정합니다.

  • Severity : Critical과 Warning중에 원하는 값을 선택합니다.(Threshold의 중요도 선택)

  • Threshold Map Names: 해당 임계치로 Monitoring할 대상 Group 또는 Host를 선택합니다.

  • Use Flag : 사용 여부를 설정합니다.

Edit버튼을 통해 이미 설정된 임계치 값을 변경할 수 있습니다.

  • 수정하기 원하는 Threshold name을 선택하고 Edit 버튼을 누릅니다.

5) Threshold Monitoring

임계치가 설정된 host는 임계치가 초과하면 Dashboard와 Monitor에서 해당 내용을 직관적으로 확인할 수 있습니다.

  • Top Menu의 Threshold 에 임계치를 초과한 숫자가 표시됩니다.

  • 노란색은 Warning, 빨간색은 Critical Event 입니다.

  • Monitor화면에서 설정된 임계치를 초과하는 Statistics는 Warning은 노란색, Critical은 빨간색 테두리로 blink되며 직관적으로 임계치가 초과되었음을 사용자에게 알려줍니다.

  • Blink는 임계치가 초과되고 있는 동안 지속되며 임계치 이하로 다시 떨어지면 멈추게 됩니다. 사용자가 임계치가 초과되고 있는 동안 임의로 blink를 멈추고 싶으면 벨모양의 버턴을 누르면 됩니다.

Nagios와 연동 및 Notification이 설정 되어있는 경우 지정된 사용자에게 설정된 방법으로 전달합니다.


6) Admin Setting

Process, Network Switch Device, Error Log, Parameter, 성능데이터를 효과적으로 모니터링 하기 위해서 수집 조건 및 필터를 설정할 수 있습니다.

6-1) 프로세스 모니터링 설정

이 메뉴에서 설정한 조건에 맞는 프로세스는 Target Process Monitoring 위젯에서 모니터링 가능합니다.

Add, Edit, Del 버튼을 통해 Process Setting 을 추가, 수정, 제거할 수 있습니다.

6-2) 네트워크 설정

Network Switch Device 등록 시 모니터링 가능합니다.

Add, Edit, Del 버튼을 통해 모니터링 하려는 Device 를 추가, 수정, 삭제할 수 있습니다.

  • IP : Network Switch Device의 Network IP를 등록합니다.

  • Name : 장치를 확인할 수 있도록 원하는 이름을 넣어줍니다.

  • Port : default로 161번을 사용하고, 변경이 가능합니다.

  • Community : Network장비에서 community를 확인해서 등록해줍니다.

  • Select a Use : Y / N중에 선택이 가능하고, N을 선택 시 등록은 완료되지만 모니터링을 하지는 않습니다. 설정 후 Manager 를 재시작해야 내용이 반영됩니다.

6-3) 에러 로그 필터

특정 패턴의 에러로그만 수집할 수 있도록 기본 설정이 존재합니다. 모든 에러로그 수집을 원하는 경우 비활성화 처리할 수 있습니다.

Add, Edit, Del 버튼을 통해 filter를 추가, 수정, 제거할 수 있습니다.

6-4) 파라미터 필터

특정 패턴의 파라미터 수집을 원하는 경우 설정하면 조건에 맞는 데이터만 저장됩니다.

Add, Edit, Del 버튼을 통해 filter를 추가, 수정, 제거할 수 있습니다.

6-5) 성능 데이터 필터

특정 패턴의 성능 데이터만 수집을 원하는 경우 설정하면 조건에 맞는 데이터만 저장됩니다.

Add, Edit, Del 버튼을 통해 filter를 추가, 수정, 제거할 수 있습니다.
